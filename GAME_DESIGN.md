# Game Design

Project Exodus: 제한된 행성 자원을 활용해 로켓을 개발하고, 점진적으로 태양계 밖으로 탈출하는 2D 산업 자동화 생존 전략 게임.

## Overview

장르:

* 2D 산업 자동화 + 생존 전략
* 제한 자원 기반 샌드박스
* 목표 달성형 세션 게임

핵심 목표:

> 황폐해진 행성에서 산업 기반을 구축하고, 로켓을 발사해 인근 행성을 개척한 뒤 더 먼 우주로 나아가 태양계 밖으로 탈출한다.

플레이어는 한정된 자원을 효율적으로 관리하며:

* 전력 생산
* 자동화 라인 구축
* 자원 채굴 및 정제
* 연구 개발
* 로켓 설계 및 업그레이드
* 연료와 로켓 부품 관리

을 진행한다.

최종 목표는 다음과 같다:

1. 초기 로켓 발사로 인근 행성에 도달
2. 그곳에서 추가 자원 확보
3. 더 강력한 로켓으로 태양계 탈출

---

## Core Feeling

게임이 전달해야 하는 감정:

* 제한된 자원으로 버티는 압박감
* 점차 확장되는 산업의 성취감
* 자동화 효율을 높일 때의 만족감
* 오염과 자원 고갈에 대한 긴장감
* 마지막에 태양계를 벗어날 때의 해방감

---

## Core Gameplay Loop

```text
자원 채굴
→ 전력 생산
→ 생산 시설 구축
→ 자동화 확대
→ 연구 개발
→ 고급 자원 확보
→ 로켓 부품 제조
→ 초기 로켓 발사
→ 인근 행성 자원 채집
→ 로켓 업그레이드
→ 태양계 탈출
```

---

## Victory Condition

### 승리 조건

* 초기 로켓 발사로 인근 행성 도달
* 인근 행성에서 추가 자원 확보
* 업그레이드된 로켓으로 태양계 밖으로 탈출

### 실패 조건

* 핵심 자원 완전 고갈
* 전력망 붕괴로 생산 대규모 중단
* 오염 수치 과도 상승
* 생존 자원(식량, 산소 등) 부족

## Narrative Direction

플레이어는 환경이 파괴된 행성에서 제한된 자원을 바탕으로 생존 산업을 확장한다.

초기 목표는 자급자족 기반을 마련하는 것이고, 중간 목표는 로켓을 만들고 인근 행성을 개척하는 것이다.

최종 목표는 태양계 밖으로 완전히 도망치는 것이다.

---

## Resource System

### 자원 흐름

* 초기 자원: 철, 구리, 석탄, 물
* 중간 자원: 석유, 우라늄, 정제 광물
* 고급 자원: 로켓 부품, 연료, 희귀 합금

모든 자원은 유한하며, 채굴 속도는 자원 고갈 및 설비 상태에 따라 변화할 수 있다.

연료는 소모품이다. 연료량에 따라 로켓이 행성 간 이동을 수행할 수 있으며, 로켓 부품은 로켓을 건설하고 업그레이드하는 데 필요하다.

### 주요 특징

* 자원은 단계별로 확장된다.
* 초기에는 기본 채굴과 정제에 집중.
* 인근 행성을 통해 새로운 자원을 확보하고 고급 생산으로 이어진다.

---

## Power System

전력은 생산과 자동화의 심장이다.

### 발전 단계

* 초기: 석탄 발전
* 중간: 태양광, 풍력
* 후반: 원자력

### 전력 부족 시 영향

* 생산 효율 감소
* 자동화 설비 정지
* 일부 건물 가동 중단

---

## Pollution System

공장 운영과 발전은 오염을 발생시킨다.

### 오염 효과

* 생산 효율 저하
* 채굴 속도 감소
* 일부 자원에 대한 접근 제한

### 오염 관리

* 필터 및 정화 시설 설치
* 친환경 기술 연구
* 오염 감축 장치 개발

---

## Research System

연구를 통해 기술과 생산 효율을 확장한다.

### 연구 테마

* 자동 채굴
* 효율적 발전
* 정제 및 가공 기술
* 로켓 엔진과 추진 기술
* 오염 저감 장치

---

## Building Categories

### 초기 건물

* 채굴기
* 발전기
* 저장고
* 기본 조립기

### 중간 건물

* 정제소
* 자동 조립기
* 연구소
* 연료 제조 시설

### 후반 건물

* 로켓 조립소
* 고급 발전소
* 환경 정화 시설
* 우주 탐사 기반 시설

---

## Art Direction

스타일:

* 저채도 산업 풍
* 황폐하고 거친 배경
* 명확한 UI 표현

UI 원칙:

* 간결함 우선
* 정보 전달 중심
* 불필요한 시각 요소 최소화

---

## Recommended Tech Stack

* 엔진: Godot 4
* 언어: GDScript
* 개발 환경: Visual Studio Code

---

## Coding Rules

```md
- Keep scripts under 300 lines.
- Prefer simple systems.
- Avoid overengineering.
- Use data-driven resources.
- Prioritize readability over abstraction.
- One responsibility per script.
```

# Core Development Principle

```text
작게 시작해서 완성한다.
```

가장 중요한 목표는:

* 완성 가능한 범위를 유지하는 것
* 시스템을 단순하게 유지하는 것
* 플레이 가능한 상태를 빠르게 만드는 것

프로젝트 규모 확장을 최대한 늦춘다.
